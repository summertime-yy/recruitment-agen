# v1.5 — 简历解析字段映射断裂修复报告

## 问题现象
原始文本中明确包含姓名、学历、学校等基础字段，但解析卡片中全部显示"待确认"（空值）。

## 根因分析（三个断裂点）

### 断裂点 1（根本原因）：LLM JSON 提取失败 → 全链路降级失败
**位置**: `src/lib/agent-tools.ts` → `LLMResumeParserTool.safeParseJSON()`

**机制**:
1. LLM 返回 JSON 但 `safeParseJSON` 解析失败 → 返回 `null`
2. LLM 路径失败 → `parsedFields = null`
3. 降级到纯正则模式 → `llmFields = undefined`
4. 正则模式对该简历的特殊格式（如 "人工智能硕士"、"base：珠海教育背景香港科技大学"、"GPA: 3.8/4.00"）也匹配不上
5. 最终所有字段为空 → 卡片显示"待确认"

**修复**:
- **4 级 JSON 提取策略**: 代码块提取 → 去除开场白 → 去除结尾文字 → 修复 + 重建
- **`tryHealAndParse()`**: 新增专门的 JSON 修复方法，处理尾随逗号、中文引号、全角标点、换行、控制字符
- **诊断日志**: 解析失败时输出 LLM 原始响应全文 + 前 1000 字符后 200 字符
- **成功日志**: 输出解析到的有效字段名和 JSON 对象

### 断裂点 2：学历正则只匹配"硕士研究生"不匹配"硕士"
**位置**: `src/lib/agent-tools.ts` → `extractDegreeRobust()`

原有逻辑 `/硕士研究生|master/i` 要求完整的"硕士研究生"，但简历中常见 "人工智能硕士"、"计算机硕士" 等单独使用的"硕士"。

**修复**:
- 新增独立"博士"/"硕士"上下文匹配（排除"博士生""论文""答辩""导师"等误匹配）
- 新增复合词模糊匹配 `/\\S{1,8}(?:硕士|硕士$)/` 用于"人工智能硕士"格式

### 断裂点 3：学校名贪婪匹配 + 短行过度合并
**位置**: `src/lib/agent-tools.ts` → `extractSchoolRobust()` + `mergePdfShortLines()`

**学校名贪婪匹配**: `([^\n\s,，]{2,15})(?:大学|学院)` 使用贪婪量词，对 "珠海教育背景香港科技大学" 会捕获 "珠海教育背景香港科技" 而非 "香港科技大学"。

**修复**:
- 改用 `matchAll()` + 非贪婪循环：遍历所有匹配项，过滤噪声词汇
- 新增 "教育背景" 到学校标签匹配模式
- 新增噪声过滤：排除包含"驳回""清单""格式""通过"的假学校名

**短行过度合并**: `mergePdfShortLines()` 将所有 ≤15 字符的行合并，破坏了字段值对结构。

**修复**:
- 新增 `isFieldValue()` 检测：手机号、邮箱、日期范围、`字段:值`、GPA/数字 → 直接输出不参与合并
- 新增 "教育背景""实习经历" 到章节标题识别

### 断裂点 4：Prompt 格式约束不够强
**位置**: `src/lib/llm-client.ts` → `getResumeParsePrompt()`

原有 Prompt 说"严格返回 JSON，不要包含解释"但格式约束太松散，LLM 经常违抗。

**修复**:
- 将格式指令改为 ✅/❌ 对照格式
- 明确列出禁止项：代码块、解释文字、中文标点、尾随逗号
- 给出单行 JSON 的具体示例
- 结尾三重强调："只回复 JSON，不要回复其他任何内容"
